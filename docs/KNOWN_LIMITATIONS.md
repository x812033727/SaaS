# 已知限制(KNOWN_LIMITATIONS)

刻意的範圍取捨,不是 bug。變更任一項前先讀對應模組 docstring。

## 金流 / 帳務
- **LINE Pay 只做一次性訂單結帳**(orders/POS)。訂閱續約仍走綠界定期定額 — LINE Pay 預授權(RegKey)商家申請門檻高,暫不支援。
- **訂單結帳連結改版即失效**(PEA-3 修補):結帳頁 URL 改以不可猜的 merchant_trade_no 為鍵,改版前已寄出的 LINE 訊息內「整數 id 型」付款連結一律 404;顧客輸入「我的訂單」即可重取新連結。改版前已產生的舊格式 trade_no(OD{id}T{time36})沿用可付款,但不具防枚舉強度 — 屬有意取捨,不回填改寫(避免斷已寄出連結)。
- **電子發票**:店家可在帳單頁選擇個人、公司統編、手機條碼／自然人憑證載具或愛心捐贈；固定以含稅電子發票開立，不提供紙本列印與悠遊卡／一卡通讀卡機載具。手機條碼與捐贈碼先做格式檢查，最終有效性仍以綠界／財政部回應為準。發票開立失敗不阻擋金流回調,由 cron 每小時重試;連續失敗需人工到綠界後台補開。
- **降級「多給不多收」**:降級當下即開始扣新方案月費,舊方案功能保留至原週期期末;不找零、不按比例退費。
- **定金可分批退款**(R4-B2):擁有者／管理員取消預約後可多次退款,每次填金額,累計不可超過定金;累計退滿才進終態(deposit_status=refunded),中途為 partially_refunded(仍可再退)。每次退款各發一則 LINE 通知(受 BOOKING_NOTIFY 旗標閘門)。**綠界自動退刷僅第一次走 API**:同一交易多次部分退刷是否被綠界接受無法確證,故第二次起一律轉人工引導(到綠界後台退刷後於系統確認人工退款,可填金額並累加)。stub 模式允許多次自動退款(測試用)。舊資料缺交易編號、非信用卡、原商店已更換、或退款結果不確定時亦轉人工。綠界正式信用卡請退款 API 無可授權測試環境,上線後仍需以自有低額正式交易驗收。

## 行銷 / 通知
- **行銷退訂採 opt-out 模型**(R6-B1,PDPA):既有顧客預設視為訂閱中(`marketing_opt_out_at` NULL),每則行銷推播(broadcast/生日/喚回/spend/welcome)附退訂連結(不可猜 `unsubscribe_token`),顧客一鍵退訂/復訂。退訂**只影響行銷推播**;交易性通知(建單/提醒/改期/取消/退款)為服務必要,恆送不可退。退訂者於 `eligible_customers` 單一入口即排除,不派獎勵不推播。退訂狀態不影響 `line_followed`(兩者獨立)。

## 整合
- **Google 日曆為單向寫入**(本系統 → 店家日曆):建單/改期/取消同步事件;店家在 Google 日曆改動不會回寫。同步暫時失敗會由 outbox 指數退避補送，達重試上限後可在行事曆頁手動重新排入。**漂移偵測**(R4-B3):每小時 cron(`check_gcal_drift`)輪詢未來已同步預約,若店家在 Google 端刪除/取消/改時間,會在預約列標示「⚠ 日曆異動」+儀表板卡片提醒+email 通知店家(每筆一次),但**絕不自動改動本系統預約**——需店家自行在後台處理;之後在本系統重新同步(改期等)或店家把 Google 事件改回一致時,旗標自動清除。
- **ICS 訂閱刷新延遲**:Google 對網址訂閱的刷新間隔不可控(常見數小時以上),需要即時請用 OAuth 連結。
- **簡訊 fallback**:已支援三竹 Mitake(`SAAS_SMS_PROVIDER=mitake` + 帳密即真發;未填走 stub 只記 log);`SAAS_SMS_FALLBACK_ENABLED` 開啟後僅在 LINE 推播失敗且顧客有手機時觸發。刻意不做簡訊 outbox 重試:提醒屬時效性 best-effort,排隊補送會變過時轟炸。僅支援台灣手機號(09xx)。
- **候補釋出會保留容量**(R4-B1):發 offer 給候補者時,該名額於回應窗口內為其保留(slot.held_count),公開池與其他顧客訂不到;offeree 本人建單消耗保留、逾時或取消候補則釋放回公開池。多 slot 平行 offer 各自獨立。

## AI
- **AI 永不直接改資料**:改期/取消一律回確認按鈕,由既有含擁有者驗證的 postback 分支執行。
- **tool loop 上限 3 輪**,末輪強制收斂;內部查詢不計 AI 月額度(每則訊息仍扣 1)。
- 平台管理員可在「平台管理 → AI 設定」加密保存 MiniMax key 並立即生效；未設定時退化為規則式 StubAgent(關鍵字比對),能力有限。

## 營運
- **代管(impersonation)30 分強制過期**、不可代管 admin、不可鏈式;API Bearer 路徑不受理代管 token。
- **稽核日誌與業務同交易**:業務 rollback 時 audit 一併消失(語意正確);log 永不因寫入失敗阻擋業務。
- **E2E 冒煙為 nightly 通知性質**,不擋部署;部署門檻是主 CI(unit/integration)。
- **報表趨勢走每日預聚合**(daily_tenant_stats,cron 台北 00:40 回填近 3 天):trend_series 讀表、缺日/今天即時補;其餘摘要(booking_summary/popular_services/revenue_trend)維持即時(SQL 下推,單租戶量可承受)。歷史日回填後才反映事後異動(最多 3 天內由 cron 吸收)。
