# 已知限制(KNOWN_LIMITATIONS)

刻意的範圍取捨,不是 bug。變更任一項前先讀對應模組 docstring。

## 金流 / 帳務
- **LINE Pay 只做一次性訂單結帳**(orders/POS)。訂閱續約仍走綠界定期定額 — LINE Pay 預授權(RegKey)商家申請門檻高,暫不支援。
- **電子發票**:買受人固定為 owner email、含稅開立;統編/載具/捐贈碼未支援。發票開立失敗不阻擋金流回調,由 cron 每小時重試;連續失敗需人工到綠界後台補開。
- **降級「多給不多收」**:降級當下即開始扣新方案月費,舊方案功能保留至原週期期末;不找零、不按比例退費。
- **定金退款走人工**:逾時未付自動取消會回補名額,但已付定金的退款(顧客取消)為個案人工處理。

## 整合
- **Google 日曆為單向寫入**(本系統 → 店家日曆):建單/改期/取消同步事件;店家在 Google 日曆改動不會回寫。同步失敗不補償(記 last_error + 告警),下次異動時自然覆蓋。
- **ICS 訂閱刷新延遲**:Google 對網址訂閱的刷新間隔不可控(常見數小時以上),需要即時請用 OAuth 連結。
- **簡訊 fallback 只有介面 + stub**:未接真實簡訊商;`SAAS_SMS_FALLBACK_ENABLED` 開啟後也僅在 LINE 推播失敗且顧客有手機時觸發。

## AI
- **AI 永不直接改資料**:改期/取消一律回確認按鈕,由既有含擁有者驗證的 postback 分支執行。
- **tool loop 上限 3 輪**,末輪強制收斂;內部查詢不計 AI 月額度(每則訊息仍扣 1)。
- 無 Anthropic key 時退化為規則式 StubAgent(關鍵字比對),能力有限。

## 營運
- **代管(impersonation)30 分強制過期**、不可代管 admin、不可鏈式;API Bearer 路徑不受理代管 token。
- **稽核日誌與業務同交易**:業務 rollback 時 audit 一併消失(語意正確);log 永不因寫入失敗阻擋業務。
- **E2E 冒煙為 nightly 通知性質**,不擋部署;部署門檻是主 CI(unit/integration)。
- **報表為即時聚合**:單租戶資料量設計,無預聚合表;資料量大時報表頁變慢屬預期,屆時再引入物化。
