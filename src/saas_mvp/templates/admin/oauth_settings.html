Google Cloud 設定教學

  1. 前往 Google Cloud Console，建立或選擇公司擁有的專案。
  2. 到「API 和服務 → 程式庫」，搜尋並啟用 Google Calendar API。
  3. 到「Google Auth Platform」完成 Branding、Audience 與 Data Access。若應用仍在測試模式，請把實際測試帳號加入 Test users。
  4. 到「Clients」建立 Web application 類型的 OAuth 2.0 Client。
  5. 在「Authorized redirect URIs」逐條加入下方兩個網址，必須完全一致，不能多斜線：
  6. 建立後複製 Client ID 與 Client Secret，貼到上一區儲存。
  7. 先按「測試 Google 日曆授權」走完一次；若看到 redirect_uri_mismatch，請回 Google Console 核對網址、HTTPS、網域及尾端斜線。

正式對外商用前，請依 Google Auth Platform 顯示的要求將應用發布為 Production；若 Google 要求驗證 Calendar 權限，需完成品牌網域與權限用途審查。

一般使用者流程

  1. 平台管理員只需在本頁完成一次 LINE Login 與 Google OAuth 設定。
  2. 使用者可到「帳號設定」連結 LINE，或從登入頁使用 Google 登入；店家 owner 可到「預約行事曆」連結自己的 Google 日曆。
  3. 所有設定立即生效，使用者不需要終端機、Docker 指令或重啟服務。
