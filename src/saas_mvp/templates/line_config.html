LINE Bot 設定

第一次設定：完整教學

LINE Login 與 LINE Bot 是不同 Channel。這裡需要的是 Messaging API Channel 的 Channel secret 與 Channel access token。

  1. 建立 LINE 官方帳號並啟用 Messaging API
    前往 LINE Official Account Manager 建立或選擇官方帳號，再到「設定 → Messaging API」啟用 Messaging API，並選擇正確的 Provider。Provider 建立後不能任意更換，正式商用建議使用公司擁有的 Provider。
  2. 取得 Channel secret
    開啟 LINE Developers Console，選擇剛建立的 Provider 與 Messaging API Channel，在「Basic settings」複製 Channel secret。
  3. 發行 Channel access token
    到「Messaging API」分頁的 Channel access token 區塊發行 token。若目前先使用 long-lived token，重新發行後舊 token 會失效，記得同步回來更新。
  4. 先在本頁儲存憑證
    將 Channel secret 與 Channel access token 貼到下方表單，按「儲存並驗證」。系統只會顯示是否已設定，不會把密鑰明文顯示回網頁。
  5. 設定 Webhook URL
    回到 LINE Developers Console 的「Messaging API → Webhook settings」，貼上此店家的專屬網址：

    {{ webhook_url }}

    按 Verify，成功後開啟 Use webhook；建議同時開啟 Webhook redelivery，讓暫時失敗的事件可被重新傳送。
  6. 避免收到兩次回覆
    到 LINE Official Account Manager 的回應設定，關閉「Greeting message／加入好友歡迎訊息」與「Auto-response／自動回應」。本平台會處理 follow 與訊息事件；若想保留官方帳號管理後台的歡迎訊息，請不要同時在下方設定平台歡迎訊息。
  7. 實機驗收
    用 Messaging API 分頁顯示的 QR code 加官方帳號好友，傳送一則文字。回到本頁按「重新測試連線」，並確認 Bot 有正常回覆。

安全提醒：Channel secret 與 access token 等同 Bot 密碼，請勿貼到聊天、Issue、截圖或前端程式碼；若疑似外洩，請立即重新發行並更新本頁。
